# 语程命令参考手册

> 语程 (YuCheng) v0.4.1 — 中文母语编程语言

---

## 一、命令行用法

```
yucheng [选项] [文件]
```

### 1.1 位置参数

| 参数 | 说明 |
|------|------|
| `文件` | 要运行的源文件（`.程` 后缀） |

### 1.2 选项

| 选项 | 短选项 | 说明 |
|------|--------|------|
| `--compile` | `-c` | 编译为独立可执行文件 |
| `--standalone` | `-s` | 编译时内嵌所有依赖库（完全独立，无需 库/ 目录） |
| `--output 路径` | `-o` | 指定编译输出路径（默认与源文件同名） |
| `--run` | `-r` | 编译后即时运行生成的可执行文件 |
| `--gui` | — | 编译为 GUI 程序（无控制台窗口，仅 Windows 目标有效） |
| `--target 平台` | — | 指定编译目标平台（windows/linux/macos），默认当前平台 |
| `--lex` | — | 仅词法分析，输出 Token 列表 |
| `--parse` | — | 仅语法分析，输出 AST（抽象语法树） |
| `--repl` | `-i` | 进入交互式 REPL 模式 |
| `--eval 代码` | `-e` | 执行单行代码 |
| `--help` | `-h` | 显示帮助信息 |
| `--version` | `-V` | 显示版本号 |

### 1.3 使用示例

```bash
# 运行源文件
yucheng 程序.程

# 编译为独立 EXE
yucheng -c -s 程序.程
yucheng -c -s 程序.程 -o 输出.exe
yucheng -c -s -r 程序.程
yucheng -c -s --gui 程序.程
yucheng -c -s --gui -r 程序.程 -o 输出.exe

# 交叉编译
yucheng -c -s --target linux 程序.程
yucheng -c -s --target macos 程序.程
yucheng -c -s --target windows --gui 程序.程

# 仅词法分析
yucheng --lex 程序.程

# 仅语法分析
yucheng --parse 程序.程

# 交互式模式
yucheng --repl
yucheng -i

# 执行单行代码
yucheng -e "输出 \"你好世界\""

# 无参数时自动进入 REPL
yucheng
```

### 1.4 构建与运行

```bash
cargo build                    # 编译
cargo run -- 文件名.程          # 运行
cargo run -- --lex 文件名.程    # 词法分析
cargo run -- --parse 文件名.程  # 语法分析
cargo run -- -i                # 交互模式
```

## 二、REPL 交互式命令

REPL（交互式模式）支持以下内部命令：

| 命令 | 别名 | 说明 |
|------|------|------|
| `退出` | `exit` / `quit` | 退出交互模式 |
| `清空` | `clear` | 清空输入缓冲区 |
| `帮助` | `help` | 显示帮助信息 |

REPL 提示符：
- `语程> ` — 正常输入提示
- `... ` — 续行提示（语句未完成时）

## 六、语句语法

### 6.1 变量声明

```
设 名字 = 表达式
常量 名字 = 表达式
变量 名字 = 表达式
```

### 6.2 条件语句

```
如果 条件:
    语句块
否则如果 条件:
    语句块
否则:
    语句块
结束
```

### 6.3 循环语句

```
// 条件循环
循环 条件:
    语句块
结束

// 遍历列表
遍 元素 于 列表:
    语句块
结束

// 遍历字典（键值对）
遍 键, 值 于 字典:
    语句块
结束

// 范围遍历
遍 i 于 范围(起始, 结束):
    语句块
结束
```

### 6.4 函数定义

```
// 命名函数
函程 函数名(参数1, 参数2 = 默认值):
    语句块
结束

// 异步函数
异步 函程 函数名(参数):
    语句块
结束

// 生成器函数
生成器 函程 函数名(参数):
    语句块
结束

// 内联匿名函数（单行返回形式）
函数(参数): 返回 表达式
```

### 6.5 异常处理

```
尝试:
    语句块
捕获 错误:
    语句块
最后:
    语句块
结束

抛出 异常值
```

### 6.6 模式匹配

```
匹配 表达式:
情况 值1:
    语句块
情况 值2:
    语句块
默认:
    语句块
结束
```

### 6.7 导入库

```
导入 "库名"
```

### 6.8 注释

```
// 单行注释
/* 多行注释 */
```

## 九、编译流程

语程编译器采用三阶段流水线：

```
源代码 (.程) → 词法分析器 (Lexer) → Token 序列 → 语法分析器 (Parser) → AST → 解释器 (Interpreter) → 执行结果
```

| 阶段 | 组件 | 输入 | 输出 |
|------|------|------|------|
| 1 | Lexer | 源代码字符串 | Token 序列 |
| 2 | Parser | Token 序列 | AST（抽象语法树） |
| 3 | Interpreter | AST | 执行结果 |

使用 `--lex` 可查看阶段 1 输出，使用 `--parse` 可查看阶段 2 输出。

### 9.1 编译为独立可执行文件

使用 `--compile` / `-c` 可将 `.程` 源文件编译为独立可执行文件：

```
源文件.程 + stub (壳程序)
        ↓  yucheng -c -s 源文件.程
复制 exe + 追加源码 + 追加依赖库 + 魔数标记
        ↓
源文件.exe (约 6 MB，零依赖)
        ↓  双击运行
检测到附加数据 → 读取内嵌源码 → 解释执行 → 输出结果
```

**参数说明**：

| 参数 | 说明 |
|------|------|
| `--compile` / `-c` | 启用编译模式 |
| `--standalone` / `-s` | 自动内嵌所有依赖库，产出完全独立的可执行文件 |
| `--output 路径` / `-o` | 指定输出路径（默认与源文件同名，Windows 为 .exe，Linux/macOS 无后缀） |
| `--run` / `-r` | 编译后即时运行（交叉编译时无效） |
| `--gui` | 编译为 GUI 程序（无控制台窗口，仅 Windows 目标有效） |
| `--target 平台` | 指定编译目标平台：`windows` / `linux` / `macos`，默认当前平台 |

**编译示例**：

```bash
# 独立编译（推荐！自动内嵌所有依赖库）
yucheng -c -s 程序.程

# 编译带依赖的程序
yucheng -c -s 数学示例.程
# 输出：已内嵌 2 个依赖库文件

# 编译 GUI 程序（无控制台窗口）
yucheng -c -s --gui GUI窗口_登录界面示例.程
# 输出：GUI 程序（无控制台窗口），已内嵌 8 个依赖库文件

# 编译后即时运行
yucheng -c -s -r 程序.程

# 完整组合：独立+GUI+即时运行+指定输出
yucheng -c -s --gui -r 程序.程 -o D:\发布\程序.exe

# 交叉编译：在 Windows 上编译 Linux 程序
yucheng -c -s --target linux Linux示例/01_基础窗口.程
# 输出：01_基础窗口 (Linux ELF 可执行文件，约 8 MB)

# 交叉编译：在 Windows 上编译 macOS 程序
yucheng -c -s --target macos macOS示例/01_基础窗口测试.程
# 输出：01_基础窗口测试 (macOS Mach-O 可执行文件，约 8 MB)

# 仅编译（不内嵌库，运行时需 库/ 目录）
yucheng -c 程序.程
```

**原理**：采用"壳程序"技术——将 `.程` 源码和所有依赖库追加到 stub 可执行文件末尾。运行时自动检测附加数据，直接解释执行内嵌源码。`--gui` 参数修改 PE 头子系统为 WINDOWS（而非 CONSOLE），使双击运行时不弹出控制台黑框窗口。`--target` 参数指定目标平台，使用对应平台的 stub 文件实现交叉编译。

**交叉编译**：在任意平台上编译其他平台的程序，需要将目标平台的 stub 文件放到语程目录或 `stubs/` 目录下：

| 目标平台 | stub 文件名 | 获取方式 |
|---------|------------|---------|
| Windows | `yucheng-windows.exe` | 在 Windows 上 `cargo build --release --features gui` |
| Linux | `yucheng-linux` | 在 Linux 上 `cargo build --release --features gui` |
| macOS | `yucheng-macos` | 在 macOS 上 `cargo build --release --features gui` |

编译出的 Windows exe 约 6 MB，零 DLL 依赖，可在任意 Windows 电脑直接运行。Linux/macOS 可执行文件约 8 MB。

## 十、Feature Flags

| Feature | 默认启用 | 说明 |
|---------|----------|------|
| `gui` | 否 | 启用 GUI 函数（Windows: Win32 原生 API 83 个内部函数；Linux: GTK4 83 个；macOS: Cocoa 73 个；35 个组件库），自动加载中文字体 |

启用方式：

```bash
cargo build --features gui
cargo run --features gui -- 文件名.程
```
